What is this?

CharityCoin is a free, open, crowd-sourced directory linking charities to their Bitcoin wallets. It's like onename or keybase, but the information is compiled, organized, and verified by people like you.

Why on earth would anyone want to do this?

I think Bitcoin and nonprofits are a natural fit, and I'm not alone. Cryptocurrencies like Bitcoin enable fast and efficient transfers of money. No credit card processing fees, no waiting days for ACH transfers to clear. Some nonprofits publish their bitcoin wallet addresses, but that information is currently not available from a reliable, verifiable source.

And then what?

Once this data is in a structured format, there are a lot of cool things that can be done with it. Really inventive APIs or services could be built with this information.

How do you verify bitcoin addresses?

Right now, a simple python script grabs the markup from the url listed in the "confirmation" field in each entry. The script checks that the address is present in the markup. This script is run on TravisCI for every pull request and every commit to master. I verify that the domain matches the domain listed for the nonprofit on guidestar.

How can I get involved?

There are 1.3 million registered nonprofits, and a growing number accept bitcoin. This is not a job for one person. The most direct way to help is to submit an issue or (for the nerds) submit a pull request for an organization that should be listed but isn't.

If you want to make a technical contribution or build something with this data, please do, and let me know how I can help. I'm spending 12 weeks launching this project. If you'd like to know more, and keep up with progress, subscribe to my tinyletter.
